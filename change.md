# Current release summary

**Version:** 3.7.0  
**Date:** 2026-04-18

## What changed

- **v3.7.0 â€” SSO managed from the admin.** Introduces an `auth_config` table and `getEffectiveAuthConfig()` helper that merges DB overrides over env defaults. New endpoints: public `GET /api/auth/config` (feeds the login UI); admin `GET/PATCH /api/admin/auth/config` for tenant id / client id / audience / issuer / allowed domains / admin role names / admin emails, plus `POST /api/admin/auth/test-sso` to dry-run a real Entra token. `resolveOwner()` and admin checks are now DB-aware (`isAdminFor(db, owner)`) so config changes take effect without restarting the server. Env vars remain the safe fallback / seed.
- **v3.6.0 â€” Local admin accounts (back-end).** New local-accounts table (`local_users`) with bcrypt-hashed passwords; short-lived HS256 JWTs (`LOCAL_JWT_SECRET`, 8 h default) feed straight into `resolveOwner()`, so every downstream endpoint accepts local sessions alongside Entra tokens. Admin CRUD for users and roles at `/api/admin/users` + `/api/admin/roles`; admin-initiated password reset (`/api/admin/users/:id/reset-password`) forces change on next login. Lockout after 5 bad attempts for 15 min; every auth/user/role change writes to `auth_audit` (visible at `/api/admin/auth/audit`). The last active `comp-plan-admin` cannot be deleted, deactivated, or demoted. First-run bootstrap admin is env-driven (`BOOTSTRAP_ADMIN_EMAIL` / `BOOTSTRAP_ADMIN_PASSWORD`).
- **v3.5.0 â€” UX polish.** Header shows **"Logged in as: {name}"** + **Sign out** + **Admin Console** (moved from footer). Save / Submit redirect to the **Submissions** tab (renamed from *Library*). Submitted records are now genuinely read-only (Tiptap + department combobox). *Reopen for editing* renamed to **Edit**. Composer action buttons are replicated at the **top** of the form. Server PDFKit fallback now mirrors the Print document layout so **Download PDF** and **Print document** produce the same output. Admin console **Print document** fixed (the hidden print layer is portalled outside `.admin-shell`).
- **Sprint 5 (v3.4.0) â€” Admin on the server.** New `/api/admin/submissions` endpoints list/patch all users' rows, guarded by **`ADMIN_ROLE_NAMES`** (default **`comp-plan-admin`**) **or** **`ADMIN_EMAILS`**. `admin.html` now requires MSAL sign-in + admin role, shows per-submission **`ownerEmail`** and status pills, and falls back to seeded localStorage with a notice when the API is unavailable. Auth context also carries **`roles`** (JWT claim or **`X-User-Roles`** header).
- **Sprint 4 (v3.3.0) â€” API JWT.** Server can validate **Entra access tokens** (`Authorization: Bearer`) using **`AZURE_TENANT_ID`** / **`AZURE_AUDIENCE`**; optional **`ALLOW_HEADER_IDENTITY=true`** keeps **`X-User-*`** trusted when migrating or before API scope consent. Client acquires **`VITE_API_SCOPE`** (default **`access_as_user`**) via MSAL silent token when configured.
- **Sprint 3 (v3.2.0) â€” Lifecycle.** Submissions carry **`draft` / `submitted`** status and **`submittedAt`**. API supports transitions and blocks deleting submitted rows. Composer: preview before submit, reopen, PDF download and email summary from the library.
- **Sprint 2 (v3.1.0) â€” Persistence.** Main app submissions live in **SQLite** via **`/api/submissions`** (Fastify). Routes: **`/app`** home (table), **`/app/compose`** composer. **`admin.html`** still uses localStorage until wired.
- **Sprint 1 (v3.0.0) â€” Entra ID + routes.** Public **landing** with **Sign in with Microsoft** (Azure Entra); **only @cabq.gov** (or `VITE_ALLOWED_EMAIL_DOMAINS`) may use the app after sign-in. See [`.env.example`](.env.example) and [`docs/VERSIONING.md`](docs/VERSIONING.md).
